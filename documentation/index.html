UML Creator

Popis

Webová aplikace pro vytváření UML diagramů

Uplikace umožňuje drag and drop manipulovat s UML elementy a drag and drop přidávat elementy.

UML elementy lze kreslit pomocí myši nebo dotikové obrazovky. Jeden tah se bere jako jeden element (u myši dokud držím stisknuté tlačítko)

Aplikace je napsaná v reactu a je objektově orientovaná, díky tomu lze jednoduše přidávat nové elemnty.

Ukázka aplikace

Aplikaci si lze vyzkoušet na adrese nejaka adresa a nebo se můžete kouknout na video s její ukázkou

Zdrojový kód (1)

Zdrojové kódy naleznete na GitHubu po stažení je potřeba nainstalovat závislostin pomocí
npm install
a pak jí lze spustit pomocí
npm start

Zdrojový kód (2)

Struktura projektu

Projekt je rozdělen do několika částí

  • src/componemts - vizuální prvky
  • src/Services - například komunikace se serverem
  • src/actions - obalují jednotlivé akce - přesun elementu, přidání element atd..
  • src/elements - obsahuje jednotlivé UML elementy (každý má svoji vlastní složku) a předek všech komponent (baseElement) od kterého dědí a který obsahuej většinu funkcionality ohledně jednotlivých prvků
  • src/reducers - metoda pro práci s reduxem (pridává/upravuje věci v redux storage)

Zdrojový kód (3)

UML elementy

Každý UML element se skládá ze 2-3 component. Jedna je pro jeho zobrazení, druhá pro jeho náhled při drag and drop přídávání a třetí nepoviná může být pro formuláž s její editací

Dále každý UML element má svojí service, která se stará o jeho rozpoznání

Zdrojový kód (4)

UML elementy - Vytváření nového

Potřeba extendovat BaseUmlElement.js a implementovat potřebné metody, extendovat BaseUmlElementPreview.js a implementovat potřebné metody a nakonec implementovat rozpoznávání.

Rozpoznávání dostane bo nakresleného obrazce a spočítá vzdálenost na druhou od nejbližšího bodu danného elementu

Nový element je potřeba přidat do metod v baseElement/services/ElementService.js

Zdrojový kód (5)

UML css

CSS soubory se nacházejí ve složce src/css kde jsou ty které se přiřazují přímo k elementům a potom je ješte jeden přímo ve root složce.

Pokročilejší selectory a transformace/animace jsou použity pro vykreslení loading spineru

Zdrojový kód (6)

Offline/online, historie

Aplikaci může po načtení běžet offline, akorát nelze uložit data (tlačítko save rozpozná, že je uživatel offline a disabluje se.

Provedenou akci lze vrátit zpět buď pomocí tlačítka, nebo pomocí prohlížeče a navigace dopředu/zpět
